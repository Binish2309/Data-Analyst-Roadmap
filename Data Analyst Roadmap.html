What Does a Data Analyst Do?

Data analysts collect, clean, analyze, and visualize data to help organizations make better decisions.

🔍

Data Collection & Cleaning

Gather raw data from databases, spreadsheets, and APIs. Clean inconsistencies, handle missing values, and prepare data for analysis.

Core Skill
📊

Find patterns, trends, and anomalies in datasets using statistics and visualization to uncover hidden insights.

The Complete Roadmap

Follow these phases in order. Click any phase to expand detailed guidance. Each phase builds on the previous — don't skip foundations!

Phase 01
Foundation Mindset & Setup
⏱ 1–2 weeks
Growth MindsetGoal SettingTools InstallationLearning Plan
  • Understand the role: Research what data analysts actually do day-to-day. Read job descriptions on LinkedIn and Glassdoor.
  • Set a timeline: Commit to 1–2 hours/day minimum. Most people land entry roles within 6–12 months of consistent learning.
  • Install your toolkit: Download Python (Anaconda), VS Code, and create free accounts on Kaggle, Google Colab, and GitHub.
  • Build a learning journal: Document what you learn each day — this doubles as portfolio evidence later.
  • Join communities: Reddit r/dataanalysis, Kaggle forums, and LinkedIn groups for accountability and support.
Phase 02
Excel & Spreadsheet Mastery
⏱ 2–3 weeks
ExcelGoogle SheetsPivot TablesVLOOKUPCharts
  • Why Excel first? Most businesses still rely heavily on Excel. It's the fastest way to get comfortable with data.
  • Learn: Formulas (SUM, IF, COUNTIF, VLOOKUP, INDEX-MATCH), Pivot Tables, Data Validation, Conditional Formatting.
  • Practice: Download 3 free datasets from Kaggle and analyze them purely in Excel before moving on.
  • Google Sheets: Learn the same skills in Sheets — it's free and collaborative, great for portfolio projects.
  • Mini project: Build a sales dashboard with charts and pivot tables using any publicly available dataset.
Phase 03
SQL — The Core Language
⏱ 4–6 weeks
SELECT / WHEREJOINsGROUP BYSubqueriesWindow FunctionsCTEs
  • SQL is non-negotiable. It appears in ~90% of data analyst job descriptions. Treat it like your professional language.
  • Start with: SQLZoo (free), Mode Analytics SQL Tutorial, or Khan Academy's SQL course.
  • Master: SELECT, WHERE, GROUP BY, ORDER BY, all JOINs (INNER, LEFT, RIGHT, FULL), subqueries, CTEs, Window Functions (ROW_NUMBER, RANK, LAG, LEAD).
  • Practice databases: Use free MySQL/PostgreSQL on db-fiddle.com or install SQLite locally.
  • Real dataset practice: Practice on Kaggle SQL courses which use real BigQuery data.
  • Project: Write 20+ queries answering business questions from a database of your choice. Document them on GitHub.
Phase 04
Statistics & Math Foundations
⏱ 3–4 weeks
Descriptive StatsProbabilityDistributionsHypothesis TestingCorrelation
  • You need to understand mean, median, mode, standard deviation, variance — these come up every day.
  • Learn: Probability basics, normal distribution, skewness, hypothesis testing (p-values, t-tests), correlation vs causation.
  • Free resources: Khan Academy Statistics (start here), StatQuest on YouTube (excellent visual explanations).
  • Don't overthink: You don't need calculus. Focus on applied statistics, not theory.
  • Apply it: Perform a basic EDA (Exploratory Data Analysis) on a dataset and describe what the statistics tell you.
Phase 05
Python for Data Analysis
⏱ 6–8 weeks
Python BasicsPandasNumPyMatplotlibSeabornJupyter
  • Start with Python basics: Variables, loops, functions, lists, dictionaries, file I/O. Use freeCodeCamp or CS50P.
  • Pandas: The most important library. Learn DataFrames, Series, read_csv, filtering, groupby, merge, apply, and handling missing data.
  • NumPy: Arrays, array operations, broadcasting — foundational for all data work.
  • Visualization: Matplotlib for basics, Seaborn for beautiful statistical plots. Practice building histograms, scatter plots, heatmaps, box plots.
  • Jupyter Notebooks: Your daily workspace. Learn to write clean, documented notebooks.
  • Projects: EDA project on a Kaggle dataset — clean data, analyze, visualize 10+ insights, publish notebook publicly.
Phase 06
Data Visualization & Dashboards
⏱ 3–4 weeks
Tableau PublicPower BIPlotlyDashboard DesignStorytelling
  • Tableau Public is FREE and the industry leader. Build 3–5 interactive dashboards and publish them — this is your most visible portfolio work.
  • Power BI Desktop is also free. Very popular in corporate environments, especially in finance and operations.
  • Plotly/Dash: For Python lovers — build interactive web-based charts.
  • Design principles: Less is more. Choose chart types intentionally. Use color purposefully. Learn the Gestalt principles of visual perception.
  • Storytelling: Every dashboard should answer a specific business question. Practice narrating your visualizations out loud.
  • Project: Rebuild a famous public dashboard and add your own insights. Share on LinkedIn with explanation.
Phase 07
Portfolio Building
⏱ Ongoing from Phase 4
GitHubKaggle ProfileLinkedInTableau PublicCase Studies
  • Your portfolio is your degree substitute. 3–5 strong projects beat a CS degree for many employers.
  • GitHub: Every project gets a GitHub repository with a clear README explaining the problem, approach, and findings.
  • Project ideas: Sales analysis, customer churn analysis, COVID data analysis, sports statistics, financial market trends, social media sentiment.
  • Quality over quantity: 3 deep, well-documented projects > 10 shallow ones.
  • LinkedIn: Post about each project. Write a 3-paragraph post explaining what you learned. This builds visibility.
  • Kaggle competitions: Participate in beginner competitions. Even finishing in the top 50% shows initiative.
Phase 08
Job Search & Applications
⏱ 4–8 weeks
Resume WritingLinkedIn OptimizationJob BoardsNetworkingCold Outreach
  • Resume: One page, ATS-optimized. Lead with a skills section showing SQL, Python, Excel, Tableau/Power BI. Quantify everything ("Reduced reporting time by 40%").
  • LinkedIn: Optimize your headline (e.g. "Aspiring Data Analyst | SQL · Python · Tableau"). Add all projects. Post content weekly.
  • Job boards: LinkedIn Jobs, Indeed, Glassdoor, AngelList (for startups), Wellfound, company careers pages directly.
  • Target: Apply for "Data Analyst", "Junior Data Analyst", "Business Analyst", "Reporting Analyst", "Marketing Analyst" — these are all entry points.
  • Networking: Message 5 data professionals per week. Not asking for jobs — asking for advice. 60% of jobs come through referrals.
  • Volume + quality: Aim for 10–15 quality applications per week with tailored cover letters.
Phase 09
Continuous Growth (Post-Hire)
⏱ Ongoing career
Advanced SQLdbtCloud (AWS/GCP)Machine Learning BasicsSoft Skills
  • Once hired, focus on deepening domain expertise in your industry (e.g. fintech, healthcare, e-commerce).
  • Learn dbt (data build tool) for data transformation — it's becoming a standard in modern data stacks.
  • Explore cloud data platforms: Google BigQuery, AWS Redshift, Snowflake.
  • Dabble in ML: Linear regression, clustering — you don't need to become an ML engineer, but understanding models makes you more valuable.
  • Soft skills matter enormously: stakeholder management, business acumen, and presenting to executives will define your career growth.

Interview Preparation

Common data analyst interview questions with detailed answers. Expand each to study.

🗄️ SQL Questions

What is the difference between INNER JOIN and LEFT JOIN? ▼
An INNER JOIN returns only rows where there is a match in both tables. A LEFT JOIN returns all rows from the left table and matching rows from the right table — if no match exists, NULLs fill the right-table columns. Use INNER JOIN when you only care about matched records. Use LEFT JOIN when you need to see all records from the primary table regardless of whether a match exists.
What are Window Functions? Give an example. ▼
Window functions perform calculations across a set of rows related to the current row — unlike GROUP BY, they don't collapse rows. Example: ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) assigns a rank to each employee within their department by salary. Common window functions: ROW_NUMBER, RANK, DENSE_RANK, LAG, LEAD, SUM OVER, AVG OVER.
How do you find duplicate records in a table? ▼
Use GROUP BY with HAVING COUNT(*) > 1:
SELECT email, COUNT(*) FROM users GROUP BY email HAVING COUNT(*) > 1. This returns all emails that appear more than once. To find the full duplicate rows, wrap this in a subquery or use a CTE with ROW_NUMBER.
What is a CTE and when would you use it? ▼
A CTE (Common Table Expression) is a temporary named result set defined with WITH before the main query. Use it to: (1) break complex queries into readable steps, (2) reference the same subquery multiple times, (3) write recursive queries. CTEs improve readability dramatically and are preferred over deeply nested subqueries in modern SQL.

📊 Statistics & Analytical Thinking

What is the difference between mean, median, and mode? When do you use each? ▼
Mean is the arithmetic average — use it for normally distributed data. Median is the middle value — use it when data is skewed or has outliers (e.g. income distributions). Mode is the most frequent value — use it for categorical data or finding the most common occurrence. A key insight: if mean ≠ median, your data is skewed. Always report which measure you used and why.
How would you explain a p-value to a non-technical manager? ▼
A p-value tells you: "If there were no real effect, what's the probability of seeing results this extreme just by chance?" A p-value of 0.03 means there's only a 3% chance the result is due to random noise. We typically use 0.05 (5%) as the threshold — if p < 0.05, we say the result is statistically significant. Avoid saying "the result is proven" — statistical significance means we have strong enough evidence to act, not absolute certainty.
What steps do you take when you encounter missing data? ▼
Step 1: Understand why data is missing — is it random (MCAR), related to other variables (MAR), or systematically missing (MNAR)? Step 2: Quantify — how many rows/columns are affected? Step 3: Options: (a) Remove rows if <5% missing and random, (b) Impute with mean/median/mode for numeric data, (c) Use forward/backward fill for time series, (d) Flag missing as a separate category for categorical data. Always document your decision and its potential impact on results.

💼 Behavioral Questions

Tell me about a time you used data to influence a business decision. ▼
Use the STAR method: Situation (context), Task (what you needed to do), Action (what analysis you performed), Result (what decision was made and its impact). For portfolio projects, frame them as: "I analyzed customer churn data for a hypothetical e-commerce company. I found that customers who hadn't purchased in 30+ days had an 85% churn rate. I recommended a targeted email campaign for the 25–35 day window, which in similar industry studies showed 20% retention improvement."
How do you handle a situation where a stakeholder disagrees with your analysis? ▼
First, listen fully — they may have domain context you don't. Then: (1) Walk through your methodology transparently. (2) Ask what specific conclusion they disagree with. (3) Explore whether both views can coexist — maybe the data shows one thing but business context changes the recommendation. (4) If still disagreeing, propose additional data collection or a controlled test to resolve the question objectively. Never become defensive — your job is to find the truth together, not to "win."
How do you prioritize when you have multiple analyses to complete? ▼
I use a framework based on: (1) Business impact — what decision does this analysis unblock? (2) Deadline — what has a hard date? (3) Effort vs. value — can a quick analysis deliver 80% of the insight? I communicate my prioritization to stakeholders proactively and set expectations clearly. For complex projects I break them into phases and deliver early insights first, rather than waiting for a perfect final analysis.
